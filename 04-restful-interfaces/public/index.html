04 Restful Interfaces

This is a RESTful API for managing decks, slides, and cards. The API allows you to create, read, update, and delete resources in a hierarchical structure.

Use the Postman collection to interact with the API endpoints and explore the functionality.

API Endpoints

Method URL Description
GET /api Retrieve all decks in the collection with slide counts.
POST /api Create a new deck with the provided title and optional tags.
DELETE /api/:deck Delete a deck if it contains no slides.
GET /api/:deck Retrieve all slides in a specific deck with card counts.
POST /api/:deck Create a new slide in the specified deck.
DELETE /api/:deck/:slide Delete a slide if it contains no cards.
GET /api/:deck/:slide Retrieve all cards in a specific slide.
POST /api/:deck/:slide Create a new card in the specified slide.
DELETE /api/:deck/:slide/:card Delete a specific card.
PATCH /api/:deck/:slide/:card Update a specific card's properties.

Testing

Use the included Postman collection to test the API endpoints:

  • Import postman.json into Postman
  • Run the database migrations and seed data with npm run prisma:migrate and npm run prisma:seed
  • Start the server with npm run dev
  • Use the collection to test the endpoints
